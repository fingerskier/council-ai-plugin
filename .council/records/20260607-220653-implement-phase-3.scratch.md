## Turn 3 — product-manager (contested scope: Gap A & Gap B)

**Gap A — CUT wall-clock (option ii).** The user's two runaway fears are money (tokens) and time. But `max_turns` *already* governs time — turns are the unit the orchestrator counts precisely, each is bounded wall-time, a 12-turn cap can't run six hours. A separate wall-clock trigger solves a problem max_turns already covers, while adding the one thing we keep out: a knob whose enforcement we can't cleanly spec. Option (i) = six config surfaces + a new sampling semantic to govern a dimension already bounded → gold-plating. "Cheaply observable" is a reason it *could* return in Phase 4, not a reason a user needs it now. Smallest honest fix: ship the triggers we can fire. **Metric: named triggers = fireable triggers = 0 phantoms.** *(If a real user later reports max_turns is a poor time proxy, that's the signal to add `max_wall_clock_minutes` — logged as Phase-4 candidate.)*

**Gap B — make max_tokens consistently SOFT (back QA fully).** A hard stop on a *rough estimate* is a lie in three directions (fires early/wastes paid budget, late/overshoots, or never). The real hard wall the user cares about — total run length — is already enforced precisely by max_turns; tokens-per-turn × capped turns already contains blow-up. max_tokens should be what its comments already say: a chair-discretion "land the plane" wrap-up signal. Smallest fix: change ~3 words in the *2 wrong files* (PLAN:207, SKILL:187) — the 6 templates + 2 yamls + example are already correct ("soft ceiling; chair wraps up"). The fix flows *toward* the templates' existing truth. Reject any proposal to *build* precise mid-run token accounting to make "hard" true — engineering to honor a word the user never asked for. **Metric: one source of truth per budget field; true hard stops = exactly what's deterministically enforceable (max_turns, scratch_max_bytes).**

**Scope-line flag for the implementer:** after the cut, re-count the "four triggers" prose so it reads honestly — don't leave "four" pointing at a different four. *(Chair note: the four *triggers* — chair-done / budget / scratch-size / user-stop — survive intact; only the budget trigger's internal sub-conditions change. Verify the prose, don't assume a recount is needed.)*
