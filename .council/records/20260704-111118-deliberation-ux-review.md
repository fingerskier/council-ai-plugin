A full-roster meeting on the plugin's deliberation logic, UX, and workflow. The output is a ranked cleanup plan with two genuine blockers — the audit-commit path (blocks continued dogfood use today) and a README install contradiction (blocks the v0.1 tag).

## Recommendation

**No structural change.** Personalities-as-data, one orchestrator, chair-routes-and-synthesizes, two-tier memory, five fireable stop triggers, two mechanical gates — the deliberation design holds together and the dogfood records prove it runs. The cleanup plan, ranked:

**1. Harden the audit-commit path — blocking for continued dogfood use; lands first** (lane: security-engineer; mechanical forms: qa-engineer).
(a) Pin the audit commit in SKILL.md, both verbs, as an explicit path allowlist — `git add .council/records/<id>.md .council/records/<id>.scratch.md .council/memory/` — never `-A`, never `-am`. The settled auto-commit-on-current-branch decision is only safe because of this pathset. (b) Give `work` step 6b a failure action — halt the loop, report the escaped paths, never commit them — and replace "main working tree stays clean" with a session-start `git status --porcelain` snapshot re-checked at each turn boundary, firing on any *new* line. (c) Two cheap secret layers inside the same commit step (explicitly not a third gate): a seat-prompt line — "reference file paths; never quote credential or secret values into the scratchpad" — and a pre-commit secret-shape grep over the record and scratch archive.

**2. Fix the README, then cold-read, then tag v0.1 — blocks the tag; text, hours not days** (lane: product-manager fixes; user cold-reads and tags).
Delete the duplicated intro paragraph in "Why a council?"; make "Getting started" the sole install narrative; shrink "## Install" to a dual-host note. This is not cosmetics: "Getting started §2" and "## Install" *contradict* each other on where to install from, and the readiness bar's second clause is "reach a running `/council convene` without asking." Publish-readiness follow-up #1 (the stranger cold read) stays open until it happens against the *fixed* README.

**3. Ship the conformance script and turn the lights on for tests — highest leverage on the board; lands behind the tag without blocking it** (lane: qa-engineer).
One file, `test/council-conformance.test.js` (`node:test`, zero dependencies), run over `examples/sample-council/` and the dogfood `.council/`: both gates in both directions; closure lines anchored at column 0 (the corpus legitimately discusses the literal markers in prose — a naive grep false-positives today); a pinned grandfather list naming exactly the two pre-archive-rule records (`20260607-210234-implement-phase-2`, `20260607-213032-review-phase-2`) with a comment citing the format memory — never a silent skip-if-missing; the eight-pattern secret-shape scan (dry-run over the full corpus: zero hits, lands green); and, once 1(a) lands, the history invariant "every commit after the pinned cutoff SHA touching `.council/records|memory` touches only allowlisted paths." Plus wiring: `package.json` with `"test": "node --test test/"` and a minimal CI workflow, or at minimum a README line — today nothing anywhere says how to run the existing tests.

**4. Pin chair-routing and budget semantics, then build the two verb signals on top** (lane: staff-engineer).
Chair ruling this session: routing is **orchestrator-inline** — the orchestrator judges as the chair and writes the one-line routing entry itself; the chair is spawned only for synthesis and adjudication. Pin the entry form QA extracted from the phase-4 archive (`## Turn N — <chair> — routing` / `— adjudication`; every non-chair turn's sub-goal appears in the nearest preceding chair entry), and pin that `max_turns` counts **spawned seat turns only** — display honesty by construction (in the phase-4 dogfood, chair turns silently consumed 3 of 6 budget turns). Only then ship the signals: the `work` per-turn progress line (`Turn 3/12 — qa-engineer: <sub-goal>`) and the meeting round header (`Round N · scratchpad NN KB`) directly above the Conclude/Continue prompt. Signals, not knobs — no meeting budget field.

**5. Operational hygiene, one product story: `info` becomes the council's status page, and sessions learn to die cleanly** (lanes: product-manager for `info`; staff-engineer for the orphan and spawn-failure spec; security-engineer for the riders and read-only seats).
`info` grows an Open follow-ups section (grep `^- \[ \]` across `records/*.md` — item, owner, record id; closing stays a manual checkbox edit by the owner, pinned in one spec sentence) and a dangling-scratchpads/worktrees listing. Preflight detects a scratchpad with no matching record and offers resume-or-archive; a crashed `work` branch is labeled **unaudited** and offered archive-or-delete, never left looking mergeable; header fields the orchestrator acts on are parsed from the header block only, never last-match. Seat-spawn failure: note it under the seat's heading, continue the round, skip a twice-failed seat. Claude meeting seats spawn with a read-only agent type where the host offers one, mirroring the Codex worker/default split — the meeting guardrail becomes capability instead of persuasion (stops mutation, not disclosure). Minor: state SKILL.md's stop-trigger semantics once and reference it from the second site.

**Cut:** staff item 7 (rotate the round's starting seat) — cut by product-manager under authority the chair delegated; parked with a named trigger (see Dissents).

## Reasoning trail

- **How the round compounded instead of repeating.** Staff opened with a ranked ten-item survey and routed lanes (items 2/3 to security, 9 plus testability to QA, 4/6/8 plus the README ruling and a gold-plating pass to PM). Every seat took its lane and *changed* the items rather than echoing them — which is why one round sufficed and the user concluded without a steer.
- **QA's evidence discipline refined staff's claims twice.** QA ran both gates by hand before proposing to automate them: Gate 2 closes both directions across the whole corpus, but Gate 1 is unverifiable for the two records predating archive-don't-delete — hence a pinned grandfather list rather than a silent skip that would let every future session skip archiving. QA also partially refuted staff item 1: routing intent is *already written* in the phase-4 archive (` — routing` / ` — adjudication` headings), so the fix is pinning an existing good practice, not inventing one — and surfaced the sharper fact underneath: chair turns ate half the phase-4 budget, so `max_turns` semantics are currently an accident, not a decision.
- **Security chained staff items 3 and 5/6b into one attack, and our own records supply the near-miss.** The only pinned `git add` in SKILL.md is the worktree commit; the audit-trail commits pin no command at all, and the publish-readiness scratch archive records the untracked-`skidl.log` sweep near-miss in so many words. A prompt-steered escape from the worktree (the parked dissent's exact channel) plus an unpinned audit commit lands a payload — or a user's temporarily-pasted token — in a legitimate-looking council commit with no merge decision ever taken. QA's history check then hardened the argument: two legacy audit commits are deliberately mixed with spec files, and a swept commit is indistinguishable from a deliberate bundle after the fact — *you cannot audit intent, only paths*. That is what promotes the allowlist from hygiene to blocking, and why the history invariant must arm at a pinned cutoff SHA.
- **PM's per-verb user framing organized the UX work.** `convene` serves the curious stranger (the README *is* the product — and the cold read found a contradiction, not just duplication, escalating staff item 10 from cosmetic to tag-blocking); `info` serves the returning user (hence the status-page story unifying staff items 2 and 8); `meeting` serves a decision-owner steering (hence the size signal); `work` serves a delegator (hence the progress line, sequenced behind the semantics pin so the denominator is honest). Item 8's cost was demonstrated reflexively: this very meeting re-derived README defects that publish-readiness follow-up #1 already tracked.
- **Priority reconciliation was a chair call.** QA ranked the conformance script first on leverage; security and PM ranked the commit hardening first. The chair ordered hardening, then README/tag, then script: "guards the user's branch today" and "blocks the tag" outrank leverage, dark tests violate none of "honest, installable, doesn't leak" (PM's line), and the script's history invariant *needs* the allowlist landed first to pin its cutoff SHA — the ordering is also the dependency order.
- **Alternatives weighed and rejected.** A `meeting_budget` knob — rejected: the human is the meeting's stop trigger by design, any threshold we pick is indefensible, and per the stop-trigger principle a named-but-arbitrary cap is a false guardrail; ship the size *signal* instead. Routing-as-chair-spawn — rejected for orchestrator-inline: halves spawns per turn and makes `max_turns` mean what the user thinks; security's audit need is satisfied either way since the routing line is written regardless. Seat rotation — cut for zero dogfood evidence; this very session shows later seats refuting and refining the first speaker twice. Follow-ups ledger, statuses, item IDs, auto-injection into seat prompts — gold-plating; one grep in `info` suffices. A third gate — never even proposed: QA and security both pre-shrank their asks to fit the two-gate line unprompted, which is the scope discipline working.
- **Settled decisions honored.** Two gates only (the conformance script adds zero session process — the Gate-2 enum was built for it and has never been written); no new budget knobs; auto-commit-on-current-branch retained and made safe by the pathset rather than reopened; security's standing injection dissent re-read and deliberately kept parked (this session's items shrink its blast radius without claiming to retire it); QA's format dissent not reopened — its trigger ("records ship malformed") has not fired; every record conforms.
